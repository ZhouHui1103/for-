## 场景一: 客户的完整服务故事

- **`场景ID`**: `ADHD-CUST-001`
- **`场景名称`**: `一个客户从购买服务到完成服务交付的全过程`
- **`优先级`**: `P0`
- **`主要角色`**: `客户`, `督导师`, `支撑运营`, `培训师`
- **`初始情境`**: *一个有ADHD督导需求的客户完成了产品购买，形成了一个"订单信息"记录，状态为"待分配"。系统中已有相应的督导小队和人员资源。*
- **`核心任务与过程`**:
  1. `支撑运营`接收到新订单，创建或更新客户信息，并将订单分配给合适的督导小队，形成"服务进程"。
  2. `支撑运营`根据分配结果联系客户，创建专属的 `客户服务群`，并邀请客户、督导师及培训师入群。
  3. `支撑运营`在群内完成标准的服务介绍和Onboarding流程，并正式将服务交接给 `督导师`。
  4. `督导师`在群内接手服务后，开始根据客户需求模块制定周期性待办事项（如起床、洗漱、每日计划等），并在日常沟通中添加临时待办。
  5. `督导师`每日与客户沟通督导进展，记录"每日计划进度"，跟踪客户的todo完成情况。
  6. 在服务关键节点（第三天、结束），`支撑运营`触发运营进程，`督导师`收集"客户主观反馈"。
  7. 如果客户提出请假需求，`支撑运营`创建"客户请假记录"并调整服务时间线。
  8. 当客户请假结束后，由`白班督导师`负责更新对应`客户请假记录`的状态。
  9. `培训师`定期评估服务质量，在"服务进程质量评估"中记录评分和改进建议。
  10. 服务临近结束时，`督导师`与客户进行服务总结、沟通续费意向并引导完成最终反馈。
  11. 如果客户有续费意向，`督导师`在群内通知`支撑运营`跟进。
  12. `支撑运营`根据续费意向创建新的“复购”订单，并据此在`服务进程跟踪表`中创建一条新的服务记录。
  13. `白班督导师`在完成当前服务归档的同时，需确认新的续费服务记录已正确生成。若发现遗漏或信息错误，则联系`支撑运营`进行修正。
  14. 无论是否续费，服务自然结束后，由`白班督导师`负责将当前`服务进程`记录的状态更新为“已结束”，并填写准确的`实际结束时间`。
  15. 如果客户的最终反馈为好评，由`白班督导师`负责在当前`服务进程`记录中更新“好评”标记，并将好评截图上传至`好评续费记录表`。
- **`期望结果`**: *客户通过完整的督导服务获得了ADHD管理能力的提升，服务过程中的所有数据都被完整记录，为后续服务优化和客户续费提供了数据支撑。*
- **`验收标准`**:
    - [ ] `订单信息表`中订单状态已从"待分配"更新为"交付结束"
    - [ ] `服务进程跟踪表`的状态已由督导师更新为“已结束”，并记录了准确的实际结束时间。
    - [ ] `最小待办单元表`中包含了该客户服务期间的所有待办记录
    - [ ] `每日计划进度表`中记录了服务期间每日的进度评估
    - [ ] `客户主观反馈表`中至少包含Onboarding反馈和结束反馈
    - [ ] `服务进程质量评估表`中有培训师的定期评估记录
    - [ ] 如有请假，`客户请假记录表`中有相应记录且服务时间线已调整
    - [ ] 对于续费客户，系统中存在由`支撑运营`创建的、新的`服务进程`记录，且记录信息准确无误。
    - [ ] 对于获得好评的服务，`服务进程跟踪表`中的“好评”标记已更新，且`好评续费记录表`中已包含对应的截图记录。

## 场景二: 运营的人员与资源管理故事

- **`场景ID`**: `ADHD-OPS-001`
- **`场景名称`**: `运营负责人员信息管理和订单分配的日常工作`
- **`优先级`**: `P0`
- **`主要角色`**: `支撑运营`, `督导师`, `培训师`
- **`初始情境`**: *运营团队需要持续管理督导师和培训师的人员信息，根据订单需求和人员档期进行合理的资源分配，确保每个订单都能得到及时、高质量的服务交付。*
- **`核心任务与过程`**:
  1. 当有督导师、培训师或运营人员入职/离职时，`支撑运营`更新"人员信息"的状态和相关信息。
  2. `支撑运营`根据人员档期和组合规则（早班、晚班、周末班），将多个督导师和一个培训师组成"督导小队"。
  3. 当接收到新订单时，`支撑运营`评估各督导小队的负载情况，选择合适的小队进行分配。
  4. `支撑运营`将订单挂载到选定的小队上，创建"服务进程"，并在督导小队群中通知相关人员。
  5. 随后，`支撑运营`创建客户服务群，完成Onboarding流程，并将服务交接给督导师。
  6. 当接收到督导师的续费通知时，负责创建“复购”订单，并据此生成一条全新的`服务进程`记录。
  7. 在服务过程中，`支撑运营`监控运营进程的触发和执行，确保关键节点的反馈收集得到落实。
  8. `支撑运营`处理客户请假申请，更新请假记录，调整服务时间线。
  9. 在`督导师`将服务进程更新为“已结束”后，`支撑运营`确认并释放对应小队的资源容量（已负责数量-1）。
- **`期望结果`**: *所有人员信息得到准确维护，订单分配科学合理，资源利用率高，服务质量稳定，运营数据完整可追溯。*
- **`验收标准`**:
    - [ ] `人员信息表`中所有员工信息准确，状态实时更新
    - [ ] `督导小队表`中小队构成清晰，负载数据准确
    - [ ] 每个新订单都能在合理时间内完成分配
    - [ ] `服务进程跟踪表`中所有服务进程都有明确的小队分配
    - [ ] `运营进程表`中关键节点的运营流程都得到正确触发和执行
    - [ ] `客户请假记录表`中请假处理流程完整，时间线调整准确

## 场景三: 培训师的质量管控故事

- **`场景ID`**: `ADHD-TRAIN-001`
- **`场景名称`**: `培训师基于数据驱动进行服务质量监控和督导师指导`
- **`优先级`**: `P1`
- **`主要角色`**: `培训师`, `督导师`, `支撑运营`
- **`初始情境`**: *培训师负责一个或多个督导小队，需要通过定期的数据分析和质量评估，确保所负责小队的服务进程质量，并在发现问题时及时介入指导或提出调整建议。*
- **`核心任务与过程`**:
  1. `培训师`定期（每3天）检查负责小队下所有服务进程的"每日计划进度"推进等级，识别异常情况。
  2. `培训师`每周对负责的服务进程进行质量评估，在"服务进程质量评估表"中记录评分和改进意见。
  3. 当发现服务质量不佳或督导师表现问题时，`培训师`在督导小队群中与相关督导师沟通了解情况。
  4. `培训师`基于数据分析结果，对督导师进行针对性的指导和培训，帮助提升服务质量。
  5. 如果问题严重且督导师无法改进，`培训师`与`支撑运营`协商，提出更换督导师的建议。
  6. 当客户对服务不满或出现投诉时，`培训师`第一时间介入，安抚客户情绪，定位问题根源。
  7. `培训师`定期总结质量数据，为运营策略优化和督导师培训体系改进提供建议。
- **`期望结果`**: *通过数据驱动的质量监控，服务质量持续稳定提升，督导师能力不断改进，客户满意度保持高水平，业务可持续发展。*
- **`验收标准`**:
    - [ ] `服务进程质量评估表`中每个服务进程都有定期的质量评估记录
    - [ ] 所有质量异常情况都有对应的处理记录和改进措施
    - [ ] `督导小队群`中有培训师与督导师的指导沟通记录
    - [ ] 服务质量评分呈现稳定或上升趋势
    - [ ] 客户投诉得到及时有效处理，满意度维持在高水平
    - [ ] 督导师的服务能力和专业水平持续提升

## 场景四: 数据驱动的每日协同工作流

- **`场景ID`**: `ADHD-COLLAB-001`
- **`场景名称`**: `数据驱动的每日协同工作流`
- **`优先级`**: `P0`
- **`主要角色`**: `督导师`, `支撑运营`, `培训师`
- **`初始情境`**: *一个典型的工作日。系统中同时存在多个状态各异的服务进程：有的刚分配，有的在平稳进行中，还有的即将结束。所有角色都需要根据自己的职责，准确、及时地维护相关的在线表格，以保障业务顺畅流转和数据准确无误。*
- **`核心任务与过程` (每日工作流)**:
  1.  **`支撑运营` 的早晨与日间工作:**
        - **订单处理**: 检查 `订单信息表`，将所有“待分配”的新订单分配给合适的 `督导小队`，然后在 `服务进程跟踪表` 中创建新记录，最后在 `督导小队群` 中通知。
        - **服务启动**: 对于分配完成的新服务，负责创建客户服务群，执行Onboarding流程，并完成向督导师的交接工作。
        - **续费跟进**: 及时响应督导师的续费请求，创建新订单并生成全新的服务进程记录。
        - **请假处理**: 关注群聊中由 `督导师` 转达的请假需求，在 `客户请假记录表` 中创建记录并更新 `服务进程跟踪表` 的时间线。
        - **流程触发**: 监控服务进程的关键节点，按时在 `运营进程表` 中创建“获取反馈”等任务，并通知 `督导师`。
  2.  **`督导师` 的日间与日终工作:**
        - **待办管理**: 与客户在 `客户服务群` 沟通时，根据内容实时在 `最小待办单元表` 中创建“临时待办”，或更新已有待办的状态为“已完成”。
        - **每日复盘**: 在每日工作结束前，为自己负责的每一个客户，在 `每日计划进度表` 中创建一条当天的进度记录，并给出A/B/C评级。
        - **请假状态跟进**: 当客户结束请假并恢复服务时，由`白班督导师`负责将对应`客户请假记录`的状态更新为“结束”。
        - **服务收尾 (若发生)**:
            - **结束归档**: 当服务在今天结束，由`白班督导师`立即将 `服务进程跟踪表` 的状态更新为“已结束”，并填写 `实际结束时间`。
            - **好评记录**: 若获得好评，由`白班督导师`更新 `服务进程跟踪表` 的好评标记，并将截图上传至 `好评续费记录表`。
            - **续费处理**: 若客户有续费意向，由服务中的督导师在群内通知`支撑运营`处理。随后，由`白班督导师`负责确认新的服务进程记录是否已正确生成，并对异常情况进行上报。
  3.  **`培训师` 的周期性检查:**
        - **质量巡查**: 定期（如每日或每两天）检查所负责小队的 `每日计划进度表`，重点关注评级为C或有异常备注的情况。
        - **反馈分析**: 定期查看 `客户主观反馈表`，了解客户满意度波动。
        - **正式评估**: 每周根据 `服务进程质量评估表` 的要求，对服务进程进行正式的打分和评价。
  4.  **`支撑运营` 的日终收尾:**
        - **资源核对**: 检查当天所有由 `督导师` 标记为“已结束”的服务进程，确认无误后，在 `督导小队表` 中更新对应小队的 `已负责数量`（-1），释放资源。
- **`期望结果`**: *所有核心数据表都得到及时、准确的维护。角色之间的信息传递基于表格数据，流畅无误。督导师能清晰地追踪与自己薪酬相关的记录，运营能准确掌握资源情况，培训师能有效监控服务质量。*
- **`验收标准`**:
    - [ ] 当日新订单在2小时内完成分配并创建服务进程。
    - [ ] 所有进行中的服务，在次日早晨10点前都有前一天的 `每日计划进度` 记录。
    - [ ] 服务结束当天，`服务进程跟踪表` 的状态必须由督导师更新为“已结束”。
    - [ ] 督导师创建的续费和好评记录，数据完整、可追溯。
    - [ ] `督导小队表` 中的资源负载数量与实际情况保持一致。
