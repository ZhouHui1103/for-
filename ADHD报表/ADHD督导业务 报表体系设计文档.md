# ADHD督导业务 报表体系设计文档

**版本**: 1.1
**日期**: 2025-10-16

---

## 1. 设计总览

### 1.1 设计哲学

本报表体系旨在为ADHD督导业务提供一套清晰、高效、可落地的量化管理工具。其设计遵循以下核心原则：

- **对象驱动 (Object-Driven)**: 报表设计的核心是业务对象。每个核心报表都围绕一个关键业务对象（如`服务进程`）或业务主角（如`督导`）展开，确保数据分析紧贴业务真实运作。
- **按需建表 (On-Demand Reporting)**: 并非所有业务对象都需要独立的报表。对于那些作为配置、或其指标可在其他核心对象报表中轻松查看的实体（如`客户请假记录`可被视为`服务进程`的一部分），我们将其指标整合，避免报表冗余。
- **角色中心 (Role-Centric)**: 为每个核心业务角色提供专属的绩效视图。报表不仅呈现业务结果，更要反映个人及团队的绩效表现，并支持按“人”钻取，实现“一个人一个视图”的管理闭环。
- **指标分层 (Layered Metrics)**: 每个报表都清晰地划分**过程指标**（监控效率与健康度）、**结果指标**（衡量产出与效果）和**财务指标**（评估商业回报），使管理者能从不同维度审视业务。

### 1.2 报表层级

本体系分为两个层级：

- **L1 - 全局汇总报表**: 面向管理层，提供业务大盘的宏观视图，如“经营健康度仪表盘”。（本文档重点在L2）
- **L2 - 领域明细报表**: 面向一线管理者和执行者，提供具体业务对象和角色的详细数据分析。本文档将重点设计此层级。

### 1.3 数据源核心要求

**[闭环关键]** 为了确保本文档中所有关于“周期时长”的指标均可被计算，底层数据表设计必须遵循以下原则：

- **状态变更时间戳**: 对于任何具有状态流转的业务对象（如`订单信息`、`服务进程`），其数据表结构中**必须包含每一个状态变更的精确时间戳字段**。
  - **示例**: `订单信息表`中，应包含`创建时间`、`进入待分配状态时间`、`进入待交付状态时间`、`进入交付中状态时间`、`进入交付结束状态时间`等字段。
- **关键动作时间戳**: 对于依赖特定业务动作计算的指标，相关SOP必须规定执行者在完成动作时记录相应的时间戳。

## 2. 领域明细报表设计 (L2)

### 2.1 对象报表

以下是根据“按需建表”原则筛选出的、需要独立设计报表的核心业务对象。

#### 2.1.1 `服务进程分析表`

- **报表目的**: 全面跟踪和分析每一个客户服务的完整生命周期，从开启到结束，监控服务交付的整体健康度、效率和质量。
- **核心维度**:
  - `时间范围`: (日期筛选)
  - `督导小队`: (下拉筛选)
  - `培训师`/`督导师`: (下拉筛选)
  - `产品清单`: (下拉筛选)
- **指标定义**:
  - **过程指标**:
    - **漏斗分析**:
      - `各阶段转化率`: “待开启 -> 进程中 -> 已结束”的转化漏斗。
    - **周期时长分析**:
      - `各阶段平均时长`: “待开启 -> 进程中”、“进程中 -> 已结束”的平均耗时。
  - **结果指标**:
    - `服务总数` / `已完成数` / `进行中数`
    - `服务按时完成率`: `实际结束时间` <= `计划结束时间`的服务占比。
    - `服务平均质量分`: 关联`服务进程质量评估`的平均得分。
    - `客户平均满意度`: 关联`客户主观反馈`的平均得分。
    - `总请假天数` / `平均请假次数`: 关联`客户请假记录`。
  - **财务指标**:
    - `服务关联收入`: 通过`订单ID`关联`订单信息表`获取的`价格`。

#### 2.1.2 `订单与转化分析表`

- **报表目的**: 监控从市场到销售的转化效率，分析订单结构和来源，评估业务的财务健康状况。
- **核心维度**:
  - `时间范围`: (日期筛选)
  - `产品清单`: (下拉筛选)
  - `订单类型`: (初次/复购/赠送)
  - `来源渠道`: (官网/微信/推荐)
- **指标定义**:
  - **过程指标**:
    - **漏斗分析**:
      - `订单分配漏斗`: `待分配 -> 待交付 -> 交付中 -> 交付结束`的转化率。
    - **周期时长分析**:
      - `各阶段平均耗时`: 从“待分配”到“交付结束”的各阶段平均耗时。
  - **结果指标**:
    - `订单总数` / `订单完成数`
    - `各类型订单占比` / `各渠道订单占比`
  - **财务指标**:
    - `订单总金额 (GMV)`
    - `客单价`: **计算公式** `GMV / 订单总数`
    - `各产品线收入分布`

#### 2.1.3 `客户价值分析表`

- **报表目的**: 跟踪客户的生命周期价值，分析客户留存、复购与流失情况，为客户关系管理提供数据支持。
- **核心维度**:
  - `时间范围`: (日期筛选, 如首次服务时间)
  - `客户状态`: (新客/服务中/复购/已流失)
  - `客户属性`: (需求属性/身份属性)
- **指标定义**:
  - **过程指标**:
    - **漏斗分析**:
      - `客户生命周期漏斗`: `新客 -> 服务中 -> 复购 -> 已流失`的转化率。
    - **周期时长分析**:
      - `各阶段平均停留时长`: 客户在“新客”、“服务中”等各个状态的平均停留时间。
  - **结果指标**:
    - `各状态客户数`
    - `客户复购率` / `客户流失率`
  - **财务指标**:
    - `客户生命周期价值 (LTV)`: **计算公式** `指定周期内, 某客户所有已完成订单的金额总和`。
    - `活跃付费客户数`: **业务规则** `过去90天内有新购或续费订单的客户数`。

### 2.2 角色报表 (个人绩效视图)

角色报表的核心是构建一个以“人”为中心的绩效仪表盘。

#### 2.2.1 `督导师绩效表`

- **报表目的**: 评估督导师的服务交付质量、工作效率和客户满意度。
- **核心视图**: **默认按`督导师姓名`汇总**，支持选择单一人员查看其个人视图。
- **指标定义**:
  - **过程指标**:
    - `任务平均完成耗时`: **[数据依赖]** `最小待办单元表`中的`完成时间` - `提出时间`。
    - `服务开启响应时长`: **[数据依赖]** 需在`服务进程表`中记录`首次联系客户时间`，计算`首次联系客户时间` - `服务进程创建时间`。
  - **结果指标**:
    - `负责服务总数` / `按时完成率` (来自`服务进程`)
    - `负责任务总数` / `完成率` (来自`最小待办单元`)
    - `客户每日计划“A”级占比` (来自`每日计划进度`)
    - `客户满意度平均分` (来自`客户主观反馈`)
  - **财务指标**:
    - `负责客户续费率` (来自`服务进程`及关联的续费`订单`)
    - `绩效奖金` / `复购激励` (来自薪酬逻辑)

#### 2.2.2 `培训师绩效表`

- **报表目的**: 评估培训师的团队管理能力、服务质量监控水平和所辖团队的整体表现。
- **核心视图**: **默认按`培训师姓名`汇总**，支持选择单一人员查看其个人视图。
- **指标定义**:
  - **过程指标**:
    - `评估覆盖率` (是否对所有应评服务完成评估)
    - `问题发现-介入时长`: **[数据依赖]** 需在`服务进程质量评估表`中记录`问题发现时间`和`介入沟通时间`。
  - **结果指标**:
    - `所辖小队服务质量平均分` (来自`服务进程质量评估`)
    - `所辖督导师平均绩效` (聚合`督导师绩效表`数据)
    - `所辖小队客户满意度平均分`
  - **财务指标**:
    - `所辖小队客户整体续费率`
    - `管理奖金` (来自薪酬逻辑)

#### 2.2.3 `支撑运营绩效表`

- **报表目的**: 评估支撑运营团队的流程执行效率、资源分配合理性和关键运营动作的完成质量。
- **核心视图**: **默认按`支撑运营姓名`汇总**，支持选择单一人员查看其个人视图。
- **指标定义**:
  - **过程指标**:
    - `订单分配平均耗时`: **[数据依赖]** `服务进程`的`创建时间` - 对应`订单`的`创建时间`。
    - `请假审批平均耗时`: **[数据依赖]** `客户请假记录`的`状态变更为已批准时间` - `创建时间`。
  - **结果指标**:
    - `服务按时启动率`
    - `关键运营动作完成率` (来自`运营进程`)
    - `流程执行准确率`
  - **财务指标**:
    - `绩效奖金` (来自薪酬逻辑)
