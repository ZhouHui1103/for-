# L2-方向提案-规范

**版本**: 1.1

---

## 一、文档定位

**层级**: L2 - 信息层 (Information Layer)
**核心输入**: `L1-事实底座` 的 “上送交接键”。

**核心目的**: 提炼出“**为什么买**”的核心答案，并形成战略方向。本文档承接L1的客观事实，进行归纳、分析和提炼，旨在将零散的数据转化为结构化的、有指导意义的商业洞察，并确立“**一点发力，全盘贯通**”的战略核心。

---

## 二、填写指南

### **通用原则**
- **遵守L0标准**: 本文档所有内容的填写，都必须严格遵守 `L0-标准制定文档` 中定义的核心概念与审美标准。
- **UVP引领**: L2的所有分析都应围绕提炼出的“IP核心能力（UVP）”展开，并思考如何将其贯穿在购买理由、独特观点和目标设计中。
- **忠于证据**: 本层级的所有结论，都必须能在L1中找到直接或间接的证据支撑。每个“购买理由”都应附上证据引用。
- **高度凝练**: 从繁杂的事实中提炼出最关键的洞察。例如，“购买理由”应是用户能记住并复述的核心卖点。
- **形成共识**: 本层级的产出是后续所有工作的基础，需要与项目相关方达成共识。

### **各模块填写要点**

1.  **IP核心能力/独特价值主张 (UVP)**: 这是L2新增的核心模块，也是整个方案的“灵魂”。它旨在提炼出IP最独特、最关键、最能形成壁垒的核心能力，是实现“一点发力，全盘贯通”的基石。
2.  **用户分类与画像**: 
    - **现状用户画像**: 严格基于L1的事实，对现有用户进行客观画像描述。
    - **内容心智画像**: 基于运营目标，提炼出未来内容可以触达的目标人群。此处的分类标准应严格遵守`L0`中关于“丰满的用户画像”的“**泛**”原则，同时应考虑**不同圈层受众**（例如C端用户 vs. B端同行）。
3.  **核心购买理由清单**: L2的核心产出。需要将UVP、用户的认可点、行业特性、IP优势等，提炼成符合`L0`标准的、朗朗上口的购买理由。**注意**：任何“结果承诺”式的购买理由，都必须在本层级找到或规划其“**上下文佐证**”。
4.  **IP独特观点与适用场景**: 用于固化IP的“人设”和“灵魂”。这里的“具体表述”应有意识地体现或服务于UVP。“适用场景”的填写必须遵守`L0`标准，并体现出初步的“平台差异化”思考。
5.  **目标清单(草案)**: 将模糊的商业需求，转化为可衡量、可执行的目标。必须引入**漏斗逻辑**和“**组合拳**”思维，并明确“**问题与对应解决点**”和“**所需物料支持**”，使其具备可操作性。
6.  **异议-化解表**: 提前预判并化解用户的购买疑虑，是提升转化率的关键。

### **交接键说明**
“上送给 L3 的交接键”需要提供清晰的战略输入：
- **IP核心能力(UVP)**: `（例）可视化情绪诊断`
- **核心购买理由[]**: `（例）["不用反复回忆细节，通过表情体态就能快速定位问题", "15年行业经验，配方独家"]`
- **异议→化解[]**: `（例）["担心没效果 -> 提供7天无理由退款", "觉得价格贵 -> 强调独家配方价值"]`
- **核心目标与优先级**: `（例）核心目标：1个月内新增私域500人`
- **IP核心人设**: `（例）专业、亲切的育儿导师`

## 三、检查标准 (Checklist)

- [ ] **UVP清晰**: 是否提炼出了明确的UVP？后续的购买理由和独特观点是否都围绕它展开？
- [ ] **证据支撑**: 是否每个关键结论（特别是购买理由和异议）都有来自L1的证据？对于“结果承诺”，是否已规划其“上下文佐证”？
- [ ] **价值提炼**: “购买理由”是否足够精炼，能直接用在广告语或销售话术中？
- [ ] **目标明确**: “目标清单”是否体现了“组合拳”思维，并包含了可量化的指标和要解决的核心问题？
- [ ] **人设一致性**: “IP独特观点”的定义，是否能统一指导后续的内容创作和沟通风格？
