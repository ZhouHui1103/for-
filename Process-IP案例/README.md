# IP案例工作流使用指南 (README)

本指南旨在说明`Process-IP案例`工作流的设计理念及具体使用方法。

---

## 1. 流程的拆解逻辑

本工作流的核心设计思想是 **“一键直出”**，即用户只需提供原始输入，AI 负责在后台完成所有复杂的分析与合成工作，直接交付最终结果。

### **第一阶段：分析 (Analysis) - 后台处理**
- **目标**: 将非结构化的原始信息（如访谈录音），系统性地解构为一套结构清晰、逻辑严密的**中间分析文档 (L0-L4)**。
- **核心动作**:
    - **L0 (画像)** & **L1 (事实)**: 萃取客观事实，构建单一事实源 (SSOT)。
    - **L2 (方向)**: 基于事实提炼战略洞察，确立核心价值主张 (UVP) 和策略方向。
    - **L3 (蓝图)** & **L4 (执行)**: 将战略方向转化为可执行的战术蓝图和具体动作。
- **说明**: 此阶段由 AI 自动完成，生成的中间文档将作为底稿保留，**无需用户人工审核**。

### **第二阶段：合成 (Synthesis) - 自动交付**
- **目标**: 将第一阶段的分析结论，高效、准确地“注入”到一个标准化的、用于最终交付的**官方报告模板**中，并转换为 Word 文档。
- **核心动作**:
    - 使用 `IP案例模板.md` 作为唯一的格式标准。
    - 从L0-L4分析文档中提取对应的结论，填充到模板的相应位置。
    - 自动转换为 `.docx` 格式。


---

## 2. 具体使用的方法

请遵循以下步骤来使用本工作流：

### **第一步：发起生成指令**
1.  **准备输入**: 准备好您的客户访谈录音、会议纪要等原始材料。
2.  **发起指令**: 向AI发起指令，要求基于原始材料，直接生成最终的IP案例分析报告。
    > **指令示例**: “@XX客户录音.mp3 请直接根据这份录音，为我生成最终的IP案例分析报告（含Word）。”

### **第二步：AI 自动执行**
- AI 将在后台连续执行以下操作：
    1.  **生成 L0-L4 中间文档**（作为逻辑支撑保留在文件夹中）。
    2.  **合成标准版报告**（基于 `IP案例模板.md`）。
    3.  **转换为 Word 文档**。

### **第三步：验收交付物**
1.  **获取交付物**:
    - 您将在 `Process-IP案例/产出/` 目录下，获得最终的 `.md` 和 `.docx` 两种格式的交付物。
2.  **反馈修改 (如有)**:
    - 如果您对最终报告有修改意见，直接指出即可。AI 会负责回溯修改底层的逻辑文档，并重新生成报告。

通过遵循以上流程，我们可以确保每一次IP案例的生成，都是一个高效、直观且结果导向的过程。
