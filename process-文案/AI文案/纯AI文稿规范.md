# 纯AI文稿规范 V1.0

**版本**: 1.0
**状态**: 生效中

---

## 1. 文档定位与核心原则 (Positioning & Core Principle)

- **定位**: 本规范定义了从“选题卡片”和“原始素材”到“纯AI生成文案”的标准。此模式旨在**脱离原始录音剪辑的限制**，产出逻辑通顺、适合短视频平台传播的纯文本口播文案。
- **核心原则：逻辑重构，精神留存**:
    1.  **主体重构**: 不需要保留录音中的原话句子，而是根据录音内容进行逻辑重构和语言润色，使其更符合短视频脚本的节奏。
    2.  **精神留存**: 虽然文字是AI生成的，但核心观点、故事细节和人设语气必须严格基于原始素材和选题卡片，**严禁**脱离素材进行虚构。

---

## 2. 核心约束 (Core Constraints)

1.  **基于选题**: 文案的核心思想、Hook（开头钩子）和结尾Call to Action，必须严格执行“选题卡片”的设定。
2.  **基于素材**: 文案中的观点、案例、故事必须来自提供的录音素材。可以在措辞上进行优化（如更口语化、更精炼），但**大的事实和观点不能改动**。
3.  **无标注输出**: 最终产出的文案为纯文本格式，**不需要**标注素材来源、时间戳，也**不需要**使用特殊颜色或符号标注AI生成部分。
4.  **平台适配性**: 文案风格应符合短视频平台（如抖音、小红书）的用户习惯，逻辑通顺，节奏感强，杜绝翻译腔。

---

## 3. 结构与格式要求 (Structure & Formatting)

### 3.1 结构要求
- **逻辑结构**: 必须从附录的【爆款文案结构公式】中选择一个最合适的框架进行构建。
- **文案分段**:
    - **标题**: 对应选题卡片的选题。
    - **Hook (开头)**: 独立成段，极具吸引力，通常对应选题卡片的Hook。
    - **正文**: 根据所选结构模型进行分段，逻辑清晰，层层递进。
    - **结尾 (CTA)**: 引导互动或关注，对应选题卡片的结尾钩子。

### 3.2 格式要求
- **纯净文本**: 全文使用标准Markdown格式，不包含任何代码块、时间戳、素材文件名或特殊颜色标记。
- **语气备注**: 可在段落前使用 `(括号)` 简要提示建议的语气，如 `(语气：严肃警示)`，但不作为强制要求。

### 3.3 交付物示例

```markdown
# 选题：中产理财陷阱

(语气：警示、真诚)

本来只要付20万就能撬动200万的保单，利息还特别低，听起来是不是很完美的套利？但我最后坚决放弃了。

大家听说过保费融资吗？简单说，就是借钱买保险。银行给你低利息，你赚保单的高收益，中间吃个利息差。听起来很美，对吧？

但我差点就心动的时候，我太太拦住了我。她说了一句特别清醒的话：在中美关系这么动荡的时候，你敢借钱投资？

保费融资的利息是浮动的。如果美国加息，你要还的利息就会飙升，甚至可能超过你的收益。到时候，你不光没赚到钱，还背了一身债。

所以，理财的终局是为了安稳，而不是为了提心吊胆。如果你也在纠结要不要加杠杆，听我一句劝：安稳睡觉比什么都重要。
```

## 4. 工作流程 (Workflow)

1.  **研读选题**: 明确选题卡片中的Hook、人设和目标受众。
2.  **消化素材**: 通读相关录音素材，提取关键观点、金句和故事细节。
3.  **结构规划**: 选择合适的【爆款文案结构公式】。
4.  **AI撰写**:
    - 使用AI重写素材内容，使其更符合口播习惯。
    - 确保Hook足够抓人。
    - 确保逻辑连贯，去除口语废话。
5.  **人工/自查**:
    - 检查是否背离了选题卡片？
    - 检查是否歪曲了原始素材的观点？
    - 检查读起来是否顺口？

## 附：爆款文案结构公式

## 00 通用类
**结构：** 钩子开头 + 塑造期待 + 解决方案 + 结尾

## 01 结构一：现象危害模型
**结构：** 现象 + 危害 + 原因 + 解决办法

## 02 结构二：信任背书模型
**结构：** 炸裂式的开头 + 人设信息 + 高密度的信息盘点 + 互动式结尾

## 03 结构三：利益前置模型
**结构：** 积极结果 + 获得感 + 方案 + 互动式结尾

## 04 结构四：金句佐证模型
**结构：** 列金句 + 佐证 + 列金句 + 佐证

## 05 结构五：行业揭秘模型
**结构：** 行业揭秘 + 塑造期待 + 解决方案

## 06 结构六：利益传递模型
**结构：** 利益传递 + 强化期待 + 解决办法 + 结尾

## 07 感性观点分享格式
**结构：** 事实 + 个人感受 + 发现问题 + 引出观点 + 讲故事 + 总结观点

## 08 经典的知识分享4段结构
**结构：** 问题描述 + 问题的拆解 + 答案描述 + 答案拆解
