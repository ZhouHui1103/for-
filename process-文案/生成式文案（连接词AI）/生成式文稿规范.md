# 生成式文稿规范 V5.2 (高级混合模式)

**版本**: 5.2
**状态**: 生效中

---

## 1. 文档定位与核心原则 (Positioning & Core Principle)

- **定位**: 本规范定义了从“粗剪文稿”（输入）到“高级混合模式视频文稿”（输出）的生成标准。此模式旨在**最大化文案的最终呈现效果**，通过将少量生成内容与IP原话**无缝融合**，产出可直接用于音频合成的、最流畅自然的文稿。

- **核心原则：无缝叙事，来源可查 (Seamless Narrative, Verifiable Sources)**: 在确保**每一段IP原话的出处（素材+时间戳）都清晰可查**的前提下，允许将起“胶水”作用的生成式短语直接融入原话句子中，形成一个语法完整、逻辑顺畅的整体。

---

## 2. 核心约束 (Core Constraints)

1.  **基于选题**: 最终的文稿，其核心思想、Hook和人设落脚点，必须严格忠于“选题卡片”的设定。
2.  **忠于原意**: 可以在措辞和结构上进行优化，但绝不能扭曲或背离IP的原始观点和事实。
3.  **原始素材必须溯源**: **每一个包含IP原话的句子**，都必须清晰地标注其主要的**素材来源**和**起始时间戳**。

---

## 3. 结构与格式要求 (Structure & Formatting)

### 3.1 结构要求
- **逻辑结构**: 文稿的整体逻辑结构，应从附录的【爆款文案结构公式】中选择一个最合适的进行搭建，以保证叙事框架的成熟度。

### 3.2 格式要求：高级混合内容模型

- **时间戳要求**:
    - **主要部分溯源**: 对于一个由“生成部分”和“IP原话”共同组成的句子，时间戳应指向**该句中IP原话的起始部分**。
    - **纯生成句无时间戳**: 如果一整句话完全由AI生成（应极少使用，且仅限于必要的转场或总结），则该行**禁止**添加 `素材` 和 `时间` 字段。
- **新增：生成内容融合与标注**: 这是**V5.2规范的核心**。
    - **定义**: 为了让句子更完整、过渡更自然而生成的“补语”，通常是关联词或引导性短句。
    - **融合规则**: 生成内容可以出现在`内容`字段的**开头、中间或末尾**，与IP原话无缝融合成一个完整的句子。
    - **限制**:
        - **内容限制**: 生成内容只作为“胶水”使用，**严禁**生成包含核心观点、事实或长篇大论的“砖块”内容。
        - **字数限制**: 每个 `{{...}}` 包裹内的生成内容，**字符数（含标点）严格限制在10个以内**。
    - **标准格式**: 所有生成的内容，必须使用 **<font color="purple">紫色字体</font>**，并用双大括号 `{{...}}` 包裹。
- **保留其他标注**: 关键词(<font color="red">红字</font>)、人名(<font color="blue">蓝字</font>)、B-roll提示(`{...}`)等规范应继续使用。


## 4. 工作流程与自查清单 (Workflow & Checklist)

### 4.1 工作流程
1.  **基于粗剪**: 以符合规范的“粗剪文稿”为基础。
2.  **寻找融合点**: 找出那些可以通过添加“前言”或“后语”来使其更完整的IP原话。
3.  **注入并融合**: 在`内容`字段中，将简短的生成式短语直接添加到IP原话的前面或后面，形成一个完整的句子。
4.  **严格标注**: 将所有生成的部分，用<font color="purple">紫色字体</font>和`{{...}}`进行包裹。
5.  **通读优化**: 反复朗读整个文稿，确保所有句子都自然流畅，听起来就像一气呵成。

### 4.2 自查清单
- [ ] 标题和钩子是否足够吸引？
- [ ] 整篇文稿是否只专注一个核心命题？
- [ ] **是否忠于原始录音的核心观点、主题和事实？**
- [ ] **是否每一个包含IP原话的句子，都标注了其主要素材来源和起始时间戳？**
- [ ] **是否所有生成内容，都清晰地用紫色和`{{...}}`标注？**
- [ ] **生成内容是否严格遵守【不超过10个字符】的字数限制，且未引入新观点？**
- [ ] 错字、病句、口语化的“嗯、啊”等语气词是否已完全清除？
- [ ] 通读时，文稿是否像一篇成熟的文章一样流畅？

# 附：爆款文案结构公式

## 00 通用类
**结构：** 钩子开头 + 塑造期待 + 解决方案 + 结尾

**原理：**
口播类视频的通用型结构，可以根据具体情况对4个环节进行增删；比如在「塑造期待」后再次加入「钩子」强化期待的效果，也一些情况会直接省略结尾文案等等！通用类公式的变化会在具体的公式案例中进行解析。

---

## 01 结构一：现象危害模型
**结构：** 现象 + 危害 + 原因 + 解决办法
(目标人群+痛点/共鸣式钩子)

**案例解析：**
1.  **现象(圈定/目标人群+钩子):** 黄皮女孩买衣服太难选颜色了
    **危害:** 挑错了颜色显的脸又黑又老
    **原因:** 但皮肤颜色很多时候又是天生的
    **解决办法:** 这三种颜色不仅显白还超显气质

2.  **现象:** 很多女孩其实不胖/但是就是小腿粗
    **危害:** 穿裤子特别不好看
    **原因:** 其实跟你长期久坐有关系
    **解决办法:** 每天做这三个动作一周就能瘦一圈

---

## 02 结构二：信任背书模型
**结构：** 炸裂式的开头 + 人设信息 + 高密度的信息盘点 + 互动式结尾
(反差类钩子, 信任背书/塑造期待, 解决方案)

**案例解析：**
1.  **炸裂式的开头/吸引注意力:** 不到100元就能搞定的一些智能家居
    **信任背书:** 同事们体验后都在要链接
    **罗列相关信息:** □□品牌智能读书灯□□元
    **互动结尾:** 宝子们还有哪些宝贝评论区见!

2.  **炸裂式的开头/吸引注意力:** 还有人不知道手机拍照是可以赚钱的吗
    **信任背书:** 好多家人已经来报喜了
    **解决方案:** 将拍到的照片上传□□网站
    **互动结尾:** 关注我给生活增添更多惊喜

## 03 结构三：利益前置模型
**结构：** 积极结果 + 获得感 + 方案 + 互动式结尾
(利益性结果前置)

**案例解析：**
1.  **积极目标前置:** 家人们!我爆单啦!
    **获得感:** 上两期讲的爆款公式火了
    **方案:** □□用完之后/粉丝直接从□□变成□□,当月单个广告变现□□一条带货视频就分佣□□

2.  **获得感:** 你只要按照我以下的脚本公式去写/我相信10条之内必有一个小的爆款

3.  **价值/方案:** 为了方便记忆我总结了以下四条 (步骤1/步骤2/步骤3/步骤4) 带货视频可以展示陈述卖点1/卖点2/卖点3

4.  **口号式结尾:** 关注我带你低成本变美

## 07 感性观点分享格式
**结构：** 事实 + 个人感受 + 发现问题 + 引出观点 + 讲故事 + 总结观点

**案例解析：**
*   **事实:** 最近发生了一个什么事
*   **个人感受:** 让我产生了什么感觉
*   **发现问题:** 这背后存在一个什么问题
*   **引出观点:** 我觉得问题是什么?
*   **讲故事:** 一个能证明我观点的故事
*   **总结观点:** 所以我们应该怎样怎样

---

## 08 经典的知识分享4段结构
**结构：** 问题描述 + 问题的拆解 + 答案描述 + 答案拆解

**案例解析：**
1.  **问题描述:** 我知道你遇到了什么问题
2.  **问题的拆解:** 我已经找到了你的问题所在
3.  **答案描述:** 我来给你看看我的解决方案
4.  **答案拆解:** 为什么这个办法能解决你的问题
