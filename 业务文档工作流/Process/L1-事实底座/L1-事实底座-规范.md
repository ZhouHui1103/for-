# L1-事实底座-规范

**版本**: 1.1

---

## 一、文档定位

**层级**: L1 - 数据层 (Data Layer)
**前置输入**: `L0-标准制定文档` 中定义的 **IP画像卡**。

**核心目的**: 构建单一事实来源 (Single Source of Truth)。本文档是整个IP打造流程的基石，旨在将访谈录音中的原始、客观信息进行结构化萃取，为后续所有分析提供无偏见的数据输入。

**禁止项**: 在本层级文档中，**严禁**进行任何形式的主观总结、归纳或推断。所有字段都应是原文的直接摘录或客观分类。

---

## 二、填写指南

### **通用原则**
- **遵守L0标准**: 本文档所有内容的填写，都必须严格遵守 `L0-标准制定文档` 中定义的核心概念与审美标准。
- **忠于原文**: 所有“原话”相关字段，必须是录音内容的直接引用。
- **时间戳**: “一句原话(含时间戳)”字段必须包含在录音或文稿中对应的`MM:SS`格式时间戳，便于快速回溯查证。
- **客观分类**: 除直接引用外，其他字段（如“来源渠道”、“阶段”等）应根据预设的客观分类标准进行标记，而非个人理解。

### **各模块填写要点**

1.  **用户快照表**: 聚焦于“单个用户”的静态画像，用于构建用户分群的基础。
2.  **产品与成交路径表**: 这是L1的核心模块，用于整合关于**单个产品/服务**的所有核心事实。如果IP有多个产品，应为每个产品建立一张独立的表格。所有字段，特别是“产品定义”和“核心卖点”，都应是用户或IP口中的客观描述，而非我方分析。
3.  **情绪与决策因素库**: 收集用户在关键节点的“感性表达”和“理性判断依据”。
4.  **竞品提及表**: 捕捉用户口中自发提及的“竞品”信息，是市场定位和差异化分析的宝贵来源。

### **交接键说明**
“上送给 L2 的交接键”是本文件最重要的输出。需要将各个模块的核心信息进行初步汇总（**只汇总，不分析**），例如：
- **分群草图**: `（例）宝妈用户-母婴社群来源-解决辅食焦虑`
- **真实触点列表**: `（例）抖音视频 -> 微信私聊 -> 朋友圈培育 -> 直播下单`
- **决策Top**: `（例）朋友推荐、价格实惠、售后保障`

---

## 三、检查标准 (Checklist)

- [ ] **可追溯性**: 是否所有关键信息都包含了原文引用或时间戳？
- [ ] **客观性**: 是否所有字段都避免了主观评价和形容词？（例如，应用“高/中/低”代替“比较好”）
- [ ] **完整性**: 是否覆盖了访谈中所有与用户、产品、触点、情绪、竞品相关的核心事实？
- [ ] **交接键清晰度**: 上送L2的交接键是否足够清晰，可以让接手人快速了解事实全貌？
