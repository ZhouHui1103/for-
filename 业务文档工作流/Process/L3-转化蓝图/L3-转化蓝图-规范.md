# L3-转化蓝图-规范

**版本**: 1.0

---

## 一、文档定位

**层级**: L3 - 策略层 (Strategy Layer)

**核心目的**: 设计出“怎么买”的清晰路径，将L2的战略方向翻译成可执行的转化体系。本文档是连接“战略”与“战术”的桥梁。

**核心输入**: `L2-方向提案` 的 “上送交接键”。

---

## 二、填写指南

### **通用原则**
- **路径化思维**: 将用户的购买过程看作是一系列连续的、可设计的节点和路径。
- **体系化构建**: 不仅要设计单个转化路径，还要构建从低价到高价的产品阶梯，实现用户生命周期价值最大化。
- **可衡量性**: 为关键节点设定可追踪的指标（即使是代理指标），为未来的优化提供数据基础。

### **各模块填写要点**

1.  **漏斗节点表**: 这是整个转化路径的“骨架”。需要将L1中发现的用户真实触点，与L2定义的购买理由相结合，设计出每个环节的核心物料和期望动作。
2.  **链路SOP（精化版）**: 这是路径的“血肉”。需要详细描述每个节点的具体执行步骤、触发条件和应急预案，使其成为可直接执行的SOP。
3.  **产品进阶阶梯表**: 从“单次转化”延伸到“长期价值”，设计引导用户持续复购、升级的产品体系。
4.  **指标字典(代理版)**: 定义衡量转化效果的关键指标。在数据系统不完善的初期，使用`高/中/低`等代理指标，是务实且有效的做法。

### **交接键说明**
“上送给 L4 的交接键”需要为内容创作和落地执行提供清晰的“剧本”：
- **节点→话题/物料/话术**: `（例）承接节点 -> 欢迎语+痛点案例集.pdf`
- **节点→期望动作**: `（例）培育节点 -> 引导用户回复“1”参与直播预约`
- **产品阶梯**: `（例）9.9元体验课 -> 199元系统课 -> 1999元私教`
- **KPI代理**: `（例）直播预约人数、案例集下载率`

---

## 三、检查标准 (Checklist)

- [ ] **路径完整性**: 设计的转化路径是否覆盖了从“引流”到“复购/裂变”的完整用户生命周期？
- [ ] **逻辑自洽性**: 产品阶梯的设计是否与用户在不同阶段的需求相匹配？SOP的执行步骤是否能有效达成节点目标？
- [ ] **可执行性**: SOP的描述是否足够清晰，团队成员能否按图索骥地执行？是否明确了依赖的工具平台？
- [ ] **可衡量性**: 关键转化节点是否都定义了衡量效果的KPI（或代理KPI）？
