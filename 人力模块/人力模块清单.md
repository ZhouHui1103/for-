# 对象清单
核心：
1. 业务分配具体到业务表格了
2. 人力表格主要做纯人力结构性变化，只要不牵扯人力部分的业务逻辑全部到其他地方
3. 考虑多主体的情况 - 


基础：
1. 招募卡片：某种程度上是一种获客策略 - 带版本号
2. 人员信息：加入过公司的人，他们会以不同角色参与业务
3. 项目：实际的项目最小单位
4. 节点：具体的节点定义，节点清单完整定义
5. 签约主体：不同类型业务主体
---
Cycle：
1. 劳务关系：主体 - 节点关系（状态：入职、离职、在职）
2. 组织关系：劳务关系 + 人员 + 节点 + 项目
3. 变更日志：
4. 培训卡片：针对特定场景｜特定角色的培训SOP
5. 培训进程：一个完整的培训进程
6. 人力异常：请假、失联、Leader/被Lead不满
---
招募
1. 招募计划：进行招募活动的最基本统筹单元，也是招募活动的起点和终点
2. 招募渠道：公司进行人员招募的相应渠道，包含但不限于：实习僧、朋友圈等
3. 线索进程：加上HR微信，有意愿加入公司的外部人员。
4. 考核记录：针对技能/要求点的考核记录｜招募&培训都会用到
5. 面培资源：针对不同


绩效考核
1. 薪酬卡片：指特定角色在特定项目中的具体绩效薪酬规则，含具体参数
2. 薪酬元规则：指特定角色在特定大类型项目（研发/铺量/私域）中的绩效薪酬规则。（个税算一个特殊的元规则，同时也是特殊的项目）
3. 绩效卡片：
4. 薪酬评估：指特定人以特定角色在特定项目中在特定时间周期内的具体评估，含最终结果。同时也可以是这个角色这个周期内所有评估的总和，代表这个人这个周期内应该拿的到手薪酬
5. 绩效评估：指特定组织关系中产生的评估，存在于执行评估表中。
