## 核心任务

**实现一个能够批量生产宣传素材的自动化脚本。**

具体场景：假设你需要为一系列线上活动制作宣传图，每张图的背景、版式都相同，只有文字内容不同。你需要开发一个脚本，根据提供的文本，自动生成符合要求的图片素材。

## 详细要求

### 1. 功能要求
- **输入**: 脚本接受一个或多个文字文本作为参数。
- **处理**: 脚本能在一个预设的网页模板上，将输入文本填充到指定位置。
- **输出**: 脚本能将生成的网页内容截图，并保存为特定尺寸的图片文件。最终产出一个包含所有生成图片的文件包。

### 2. 技术要求
- **核心工具**: 必须使用 `Claude Code` 作为主要的编码和执行环境。
- **自动化浏览器**: 必须使用 `Playwright MCP` 或其他你熟悉的浏览器 `MCP (Managed Component Protocol)` 来操作网页、填充文本和截图。
- **开发流程**:
    - **网页枢纽**: 整个工作流需要以一个本地或公开的网页文件为核心。具体实现路径为：通过脚本将指定文本动态填充到该网页文件中，然后利用 `Playwright MCP` 对更新后的网页进行截图，以此方式生成最终素材。
    - **AI 方案**: 鼓励使用 `Kimi` 等大模型辅助生成 `Claude Code` 的执行方案。
    - **测试驱动**: 请在开发过程中体现测试驱动开发（TDD）的理念，确保脚本的稳定性和正确性。
- **版本控制**: 项目须使用 `Git` 进行版本管理，并将最终代码上传到你自己的 GitHub 仓库。

### 3. 交付物要求

1.  **代码仓库**: 一个公开的 GitHub 仓库链接，其中应包含以下内容：
    *   **自动化脚本**: 能够完成核心任务的脚本文件。我们鼓励实现端到端的完全自动化。
    *   **操作手册 (`manual.md`)**: 详细说明如何配置环境和运行脚本。
    *   **（可选）Claude Code 执行文档 (`cc-runner.md`)**: 如果你的方案包含部分手动步骤或需要 `Claude Code` 介入执行（例如调用 `MCP` 截图），请提供此文档。它应包含清晰的指令，能让 `Claude Code` 读取并完成整个任务流程。

## 资源与指南

### 1. Claude Code 安装与配置

1.  **全局安装**:
    在终端中执行以下命令。你可以直接在 Cursor 的终端中进行，如果遇到权限问题，它会引导你解决。
    ```bash
    npm install -g @anthropic-ai/claude-code
    ```
2.  **登录**:
    - 确保网络环境通畅（可能需要 VPN）。
    - 在终端输入 `claude` 并根据提示步骤，在网页中完成登录。
    - **账户**: `ai@x-silicon.xyz` （使用前请联系 **@大康Silicon** 开设权限）
    - **密码**: `XiGuiTech2023`
3.  **开启 YOLO 模式 (推荐)**:
    YOLO (You Only Live Once) 模式将给予 `Claude Code` 完全的自主执行权限，免去手动确认的步骤，提高效率。
    ```bash
    alias cc-yolo="CLAUBBIT=1 CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1 ENABLE_BACKGROUND_TASKS=1 claude --dangerously-skip-permissions"
    ```
    后续在终端中使用 `cc-yolo` 命令即可进入此模式。普通模式则继续使用 `claude`。

### 2. Playwright MCP 配置

1.  **添加 MCP**:
    直接在 `Claude Code` 中运行以下命令，即可添加 Playwright MCP。
    ```bash
    /mcp add playwright npx @playwright/mcp@latest
    ```
2.  **使用**:
    添加后，在 `Claude Code` 输入 `/mcp`，初次使用会显示「Connecting」状态，稍等片刻即可连接成功并使用。
