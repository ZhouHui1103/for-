# 关键概念与标准梳理

以下是在对话中被重点探讨，并为其建立了更清晰标准的核心概念：

- **清晰的卖点 (Clear Selling Point)**
    - **标准**: 必须具体、精准，最好包含数字或明确承诺，可以直接作为营销话语。要避免使用“快速”、“高效”等模糊形容词。
    - **示例**: “10分钟内帮你解决情绪问题” 是好的；“快速看见情绪问题” 是坏的。

- **清晰的产品定义 (Clear Product Definition)**
    - **标准**: 必须是一个市场可理解的品类（如课程、咨询、游学营），并清晰描述其为客户提供的核心价值。
    - **示例**: “企业服务游学营” 是好的；“企业社会服务项目” 是坏的。

- **用户分类 (User Classification)**
    - **标准**: 需要包含两种不同的视角：1. **现状用户画像**：基于真实情况，客观描述当前的核心用户是谁。2. **内容心智画像**：基于内容运营需求，提炼出的不同需求/痛点的人群，用于指导选题。
    - **示例**: 现状用户可能是“无学可上的中专生”，而内容心智用户可以是“为孩子前途焦虑的家长”。

- **计划 (Plan)**
    - **标准**: 指的是具体的、可执行的、有明确产出物的“任务清单(To-do List)”。
    - **示例**: “制作一支高质量人设故事视频” 是好的；“建立以吴老师为核心的IP账号矩阵” 是坏的（过于宏大）。

- **市场痛点 (Market Pain Point)**
    - **标准**: 必须具体，明确指出是哪个人群在什么场景下的什么具体困扰。
    - **示例**: “传统教育脱离现实，导致学生就业难” 是好的；“解决社会对职高的普遍焦虑” 是坏的（不够具体，焦虑是什么？）。

- **适用场景 (Applicable Scenarios)**
    - **标准**: 需要包含两个层面的信息：1. **渠道/物料**: 明确指出该观点适用于哪个具体的公域/私域渠道的哪种物料形式（如：抖音短视频开头、朋友圈文案）。2. **内容示例**: 提供一句具体的文案范例。

- **丰满的用户画像 (Rich User Profile)**
    - **标准**: 在为短视频等公域内容平台规划用户画像时，标准是“泛”，即尽可能多地包含所有可能对该话题感兴趣的人群，而不是过早地进行精准筛选。

- **内容方向 (Content Direction)**
    - **标准**: 需要包含至少三个维度的信息：1. **呈现方式**（如口播、测评、剧情）；2. **核心话题/选题**；3. **功能性**（即内容目标，如涨粉、转发、引流、转化）。
