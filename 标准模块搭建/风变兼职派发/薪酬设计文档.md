### 一、 指标定义

*   **工作单位**：标准化的任务工作量单元，预期为30分钟的平均工作时长。
*   **接单量**：在一个考核周期内，完成并最终通过外审的工作单位总数。
    *   对 **制作节点**，指其 **个人** 的工作单位总数。
    *   对 **培训节点**，指其所带领 **团队** 的工作单位总和（即“团队总接单量”）。
*   **弃单率**：在一个考核周期内，（主动放弃的订单数 / 总承接订单数）x 100%。这是一个负向指标，越低越好。
    *   对 **制作节点**，指其 **个人** 的弃单率。
    *   对 **培训节点**，指其所带领 **团队** 的平均弃单率。
*   **培训量**：在一个考核周期内，成功完成0-1培训并首次晋升为“S级”的组员数量。
*   **团队S级保持率**：在一个考核周期内，（月末团队S级人数 / 月初团队S级人数）x 100%。
*   **订单跟踪完成率**：在一个考核周期内，成功完成全流程跟踪（符合时间要求+完成外审）的订单比例。
*   **审核[N]次通过率**：指在某个指定的审核环节中，提交的内容在 N 次（含 N 次）内获得通过的比例。
    *   **适用说明**: 此指标可灵活应用于不同场景。例如，使用 `内审二次通过率` 考核制作节点的交付质量；使用 `外审一次通过率` 评估内部审核的质量。

### 二、 薪酬元规则 (Compensation Meta-rules)

#### **`XC-1-1`：按【工作单位】结算的服务薪酬**
1.  **规则**：根据节点完成的、并通过审核的（个人/团队）工作单位数量，计算其基础服务薪酬。
2.  **概念定义**：
    *   `核算工作单位`：用于薪酬计算的工作单位。
        *   对 **制作节点**，指其 **个人** 完成并通过审核的工作单位。
        *   对 **培训节点**，指其所带领 **团队** 完成并通过审核的工作单位总和。
3.  **参数**：
    *   `参数1`：内审通过基础薪酬（元/工作单位）。
    *   `参数2`：外审通过额外薪酬（元/工作单位）。
4.  **计算方法**：
    *   `薪酬 = (参数1 * 内审通过的核算工作单位数量) + (参数2 * 外审通过的核算工作单位数量)`
5.  **数据来源**：订单管理系统、人力资源系统-团队模块
6.  **相关注释**：此规则旨在通过“苦劳（内审）+功劳（外审）”结合的方式，保障基础收入，激励高质量交付。它统一适用于制作节点和培训节点。

#### **`XC-1-2`：S级状态激励奖金规则**
1.  **规则**：根据节点（或其团队）的S级状态，在月末发放激励奖金。
2.  **概念定义**：
    *   `S级状态核算单位`：
        *   对 **制作节点**，如果其自身在一个完整的自然月内保持S级状态，则核算单位为 `1`。
        *   对 **培训节点**，核算单位为该培训节点所带领的团队中，在月末持续保持S级的 **组员总数**。
3.  **参数**：
    *   `参数1`：每S级状态核算单位奖金（元/单位）。
4.  **计算方法**：`奖金 = 参数1 * S级状态核算单位数量`
5.  **数据来源**：人力资源系统-绩效模块
6.  **相关注释**：此规则统一适用于对制作节点（个人S级激励）和培训节点（团队S级激励）的奖励。

#### **`XC-2-1`：培训节点培训激励奖金规则**
1.  **规则**：根据培训节点完成的培训进程，对“完成基础培训”和“成功晋升S级”两种结果分别给予奖励。
2.  **概念定义**：
    *   `完成基础培训`：组员完成了所有必要的培训流程，但其最终评级未达到S级。
    *   `新晋升S级`：组员完成了所有培训流程，且其最终评级达到了S级。
3.  **参数**：
    *   `参数1`：完成基础培训薪酬（元/人）。
    *   `参数2`：新晋升S级培训薪酬（元/人）。
4.  **计算方法**：`奖金 = (参数1 * 当月完成基础培训的人数) + (参数2 * 当月新晋升S级的人数)`
5.  **数据来源**：人力资源系统-培训与绩效模块
6.  **相关注释**：此规则旨在同时激励培训的普及（基础完成）和质量（晋升S级）。

#### **`XC-2-2`：培训节点晋升激励奖金规则**
1.  **规则**：若某培训节点所带领的制作节点成功晋升为新的培训节点，则给予原培训节点一次性激励奖金。
2.  **概念定义**：
    *   `节点晋升`：指某制作节点通过考核，正式成为一名独立的培训节点。
3.  **参数**：
    *   `参数1`：每晋升一位新培训节点奖金（元/人）。
4.  **计算方法**：`奖金 = 参数1 * 当月成功晋升为培训节点的组员人数`
5.  **数据来源**：人力资源系统-晋升记录模块

#### **`XC-4-1`：项目管理订单跟踪薪酬规则**
1.  **规则**：每完整地跟踪一个订单的全流程，并确保其最终成功交付，即可获得固定薪酬。
2.  **概念定义**：
    *   `订单全流程跟踪`：从接收订单、分配任务、跟进执行进度，到最终确认甲方完成审核的完整管理过程。
3.  **参数**：
    *   `参数1`：订单全流程跟踪费（元/单）。
4.  **计算方法**：`薪酬 = 参数1 * 当月成功完成跟踪的订单总数`
5.  **数据来源**：订单管理系统

#### **`XC-3-1`：内部审核薪酬规则**
1.  **规则**：根据完成的审核次数以及该次审核的有效性（是否一次性通过外审）来计算薪酬。
2.  **概念定义**：
    *   `有效审核`：内部审核通过后，首次提交给甲方审核即获通过。
    *   `无效审核`：内部审核通过后，首次提交给甲方审核未获通过。
3.  **参数**：
    *   `参数1`：有效审核薪酬（元/次）。
    *   `参数2`：无效审核薪酬（元/次）。
4.  **计算方法**：
    *   `薪酬 = (参数1 * 有效审核次数) + (参数2 * 无效审核次数)`
5.  **数据来源**：订单管理系统-审核模块

### 四 核心指标：

本章节旨在基于已定义的薪酬规则，建立核心的财务与人力效率评估模型。

#### **1. 单节点月度到手薪酬测算**

*   **制作节点到手薪酬** = 
    ( `月度接单量` * `内审通过基础薪酬` ) + 
    ( `月度接单量` * `外审通过率` * `外审通过额外薪酬` ) + 
    `S级月度奖金 (若达标)`
    
    *   **示例**: 一位S级制作节点，月接单120个工作单位，外审通过率95%。
    *   **薪酬计算**:
        *   基础服务薪酬: (120 * 9) + (120 * 0.95 * 9) = 1080 + 1026 = **2106元**
        *   S级月度奖金: **500元**
        *   **合计薪酬**: 2106 + 500 = **2606元**

*   **培训节点到手薪酬** = 
    ( `团队总接单量` * `团队服务内审薪酬` ) + 
    ( `团队总接单量` * `团队平均外审通过率` * `团队服务外审薪酬` ) + 
    ( `月末团队S级组员数` * `S级团队保持奖金` ) +
    ( `当月完成基础培训人数` * `完成基础培训薪酬` ) +
    ( `当月新晋升S级人数` * `新晋升S级培训薪酬` ) +
    ( `当月晋升为培训节点的组员数` * `节点晋升激励奖金` )

    *   **示例**: 一位培训节点，其团队情况如下：
        *   团队规模：5名S级成员
        *   团队业绩：月总接单量600工作单位，平均外审通过率95%
        *   培训成果：本月新培训3人，其中1人完成基础培训，2人成功晋升S级
        *   人才培养：本月有1位组员成功晋升为新的培训节点
    *   **薪酬计算**:
        *   团队服务薪酬: (600 * 1) + (600 * 0.95 * 1) = 600 + 570 = **1170元**
        *   S级团队保持奖金: 5 * 200 = **1000元**
        *   培训激励奖金: (1 * 200) + (2 * 100) = 200 + 200 = **400元**
        *   **合计薪酬** = 1170 + 1000 + 400 = **2570元**
        *   **额外奖金**: 此外，该节点还将获得 `1 * 节点晋升激励奖金`。由于该项奖金参数`[待定]`，故未计入总额。

*   **项目管理节点到手薪酬** = `当月成功完成跟踪的订单总数` * `订单全流程跟踪费`

    *   **示例**: 一位项目管理节点，当月成功跟踪1000个订单，假设`订单全流程跟踪费`为2元/单。
    *   **薪酬计算**:
        *   订单跟踪薪酬: 1000 * 2 = **2000元**
        *   **合计薪酬**: **2000元**

*   **内审节点到手薪酬** = 
    ( `月度审核总次数` * `外审一次通过率` * `有效审核薪酬` ) + 
    ( `月度审核总次数` * (1 - `外审一次通过率`) * `无效审核薪酬` )

    *   **示例**: 一位内审节点，月审核500次，外审一次通过率为70%。
    *   **薪酬计算**:
        *   审核薪酬: (500 * 0.7 * 0) + (500 * 0.3 * 3) = **450元**
        *   **合计薪酬**: **450元**

#### **2. 每工作单位订单成本测算**

此处成本仅核算与订单交付直接相关的线性成本，暂不包含培训、S级奖金等非线性成本。

*   **核心逻辑**: 订单在交付过程中，由于存在弃单并被重新分配的可能，因此与“内审”环节相关的成本（包含给制作节点和内审节点的费用）会随着“弃单率”的增加而以等比数列的形式增长。成本的计算遵循 `总内审成本 / (1 - 平均弃单率) + 总外审成本` 的模型。

*   **成本构成**:
    *   `单位内审成本` = `制作节点内审薪酬` + `培训节点团队服务内审薪酬`
        *   = 9 + 1 = **10 元/工作单位**
    *   `单位外审成本` = `制作节点外审薪酬` + `培训节点团队服务外审薪酬`
        *   = 9 + 1 = **10 元/工作单位**
    *   `单次内审节点成本` = (`外审一次通过率` * `有效审核薪酬`) + ((1 - `外审一次通过率`) * `无效审核薪酬`)
        *   基于考核目标(70%)计算: (0.7 * 0) + (0.3 * 3) = **0.9 元/次** 
    *   `单次项目管理成本` = `订单全流程跟踪费` (待定)

*   **最终核算公式**:
    *   **每工作单位订单成本** = (`单位内审成本` + `单次内审节点成本`) / (1 - `平均弃单率`) + `单位外审成本` + `单次项目管理成本`
    
    *   **示例**: 假设平均弃单率为20%，项目管理成本为2元/单（即每个工作单位），则每工作单位成本为:
        (10 + 0.9) / (1 - 0.2) + 10 + 2 = 10.9 / 0.8 + 10 + 2 = 13.625 + 12 = **25.625元**

#### **3. 单个节点培训成本**

此处成本指将一名新成员培养至“S级”时，支付给培训节点的直接薪酬成本。

*   **单个S级节点培训成本** = `新晋升S级培训薪酬` = **100 元/人**
*   **单个基础培训节点成本** = `完成基础培训薪酬` = **200 元/人**

*   **说明**: 以上成本尚未包含 **生命周期人力节点(Cycle-HR)** 在招募、引导、管理过程中投入的人力时间成本。
