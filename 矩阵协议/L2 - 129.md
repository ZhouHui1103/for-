# 总览：

> 有时候会几个流程组共享一个L2文档，则在下列部分中，分别用3级标题标记编号

整体围绕项目管理、内容策略、内容分发、节点间沟通的全部流程。

  

核心模块：

1. 测试期间，每个策划都对应有一个虚拟化项目容器（文档），可以相对独立的负责，不会混淆；虚拟化的资源有：所有内容策略层内容，全部都虚拟化，内容督导也会虚拟化
    
2. 实际项目则会有一个经验沉淀池，会沉淀整体里程碑，做整体进度管理。核心管理的是中心假设（对项目的假设），一旦假设被验证，则广播至所有虚拟容器。
    
3. 所有素材生产和发布计划全部被封装到，策划只做素材审核，并且给HRBP打分反馈。
    
4. 策划在虚拟容器中（提出假设 + 提出策略 + 提出计划 + 获取数据）｜交付经理和HRBP搞定一切容器支撑的问题，一个对客户，一个对项目管理｜虚拟容器在项目层如何表达，可能对应1-2个小组，即自动化&人力运作
    
5. 交付经理核心应该是策划的HRBP，审核策划关键节点内容，对整体假设负责｜项目管理负责安排结算（后续在流程和数据里明确吧，数据应该再增加一个审核人）

## 对象总览：

> 这里列出来所有有的对象，并且一句话说明对象定义；
> 
> 对象的核心概念需要标注：状态、属性、彼此关联
> 
> 属性：短暂/长期 ｜主动指令/被动承接｜是否定期
> 
> 状态：贯穿这个变量生命周期的是怎样的状态树
> 
> 类别：这个状态的分类方式；如果有平行的话用｜隔离开
> 
> 关联字段：和哪些字段是有关联关系的

### 项目层（交付经理主导）

1. 项目： 依附单个交付经理上，可以有很多策划和督导，后续具体策划督导会分配到卡片/团上
    
    1. 属性：长续、主动指令
        
    2. 状态：待启动、测试中、铺量中
        
    3. 类别：已有模型项目、策略合作项目、内部策略项目｜电商项目、私域项目
        
    4. 关联字段：交付经理（单选｜人员信息.角色 = 交付经理）、打样策划（多选｜人员信息.角色 = 外部策划）、客户信息（单选）、顾问属性（多选｜客户自身、策划自身、外部策划）、策略顾问（多选｜人员信息.角色 = 策略顾问
        
    5. 其他字段：转化链路、备注说明
        
    6. 关联文档：内容SOP集合、项目交付日志
        
    7. 关联场域：项目合作群、项目策划群
        
    8. 关联报表：项目进展报表、项目模型测算表、项目财务报表
        
    9. 备注说明：
        
2. 结算数据（转化层）- for 客户结算，依附在整个项目上，考虑后端结算
    
    1. 属性：短暂、被动记录、二次数据、定期触发（每周）
        
    2. 状态：待核算、内部已确认、客户已确认、待结算、已结算
        
    3. 类别（结算卡片类型）：原始素材核算、有效素材核算、CPS结算、线索量核算
        
    4. 关联字段：项目、结算卡片、交付经理、客户（项目.客户）、核算数据
        
    5. 关联报表：项目财务报表
        
3. 里程碑事件：依附在某个策划负责的项目上，记录一切可以沉淀到项目可复用策略相关的事情。
    
    1. 属性：被动记录｜事件记录｜二次数据
        
    2. 日期
        
    3. 状态：已提出、交付经理验收、熙康验收
        
    4. 类别：首次转化、首次爆款、首次复现、首次模型跑正、钩子胚验收、内容卡片验收、选题方向验收｜全局首次、特定小组
        
    5. 关联字段：项目、交付经理、所属策划（人员信息.角色 = 外部策划）、审核人（人员信息.角色 = 策略督导）
        
    6. 关联报表：项目财务报表、项目进展报表、项目模型测算表
        
    7. 填写&审核&维护：
        
4. 项目测试计划：依附在某个策略督导&项目上，交付经理负责写作。
    
    1. 属性：主动指令、短期
        
    2. 状态：待提方案、待测试、已结束
        
    3. 类别：选题方向、转化方向、人设方向？（待定）、铺量策略
        
    4. 关联字段：项目、交付经理、关联策略、钩子胚、测试单元（多选）、关联团（可多选）
        
    5. 关联文档：「内容测试」
        
    6. 关联报表：项目模型测算表、
        
    7. 备注：
        
        1. 测试计划和策略调研同理｜会关联测试单元
            
        2. 铺量会对应一个长线的测试
            

  

  

### 人力层：（项目管理 Lead HR 主导）

账号小组+对应助理小队+策划&项目 + HRBP是对应到一起的（如果是系统发布则HRBP就是系统的负责人）

1. 账号小组（团）：依附在某个策划&项目上｜人力侧会挂在HRBP下｜后续落地策略相关都会挂在特定团下
    
    1. 属性：主导指令、长期｜生命周期伴随「项目」
        
    2. 状态：运作中、终止运作
        
    3. 类别：测试｜铺量
        
    4. 关联字段：项目、打样策划（单选｜人员信息.角色 = 外部策划）、HRBP（单选｜人员信息.角色 = HRBP/分发系统）、策略顾问（单选｜人员信息.角色 = 策略顾问）
        
    5. 指令字段：制作助理数量、发布助理数量、每日发布数量、账号种类（多选｜实名认证、非实名认证、关联主理人）、转化链路、设备要求
        
    6. 其他字段：制作助理、自动化负责人、发布助理（多选，直接选人）
        
    7. 关联报表：
        
2. 客户信息：简单的一个标记，后续所有客户相关结算/项目信息都会挂在其中
    
    1. 属性：被动承接、目录
        
    2. 关联字段：客户经理（单选｜人员信息.角色 = 客户经理）
        
3. 人员信息：所有需要做功的人，无论内部外部，策划&交付经理&HRBP&策略督导都会在这里登记
    
    1. 属性：被动承接、目录
        
    2. 状态：有效、无效
        
    3. 其他字段：人员、角色（策划、HRBP、交付经理、客户经理、项目管理、系统分发、产品）、外部人员、档期
        
    4. 备注：
        
        1. 档期的概念：策划 - 生命周期、交付经理 - 会议数量、
