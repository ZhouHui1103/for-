## 场景一: 客户服务生命周期

- **`场景ID`**: `MATRIX-CUST-001`
- **`场景名称`**: 客户服务生命周期
- **`优先级`**: `P0`
- **`主要角色`**: `产品&销售中心`, `测试服务中心`
- **`初始情境`**: `客户信息` 已初步建立，客户存在明确的业务需求，并已与 `产品&销售中心` 取得联系。
- **`核心任务与过程`**:
  1.  `产品&销售中心` 与客户达成合作，创建 `项目` 对象，并填充 `关联客户`、`转化链路` 等核心字段，将 `项目.状态` 设为 `待启动`。
  2.  `产品&销售中心` 将 `项目` 移交给 `测试服务中心`，并将 `项目.交付经理` 字段更新为服务中心指定的负责人。
  3.  `测试服务中心` 作为客户的唯一接口，在关键节点创建 `里程碑事件` 并同步给客户，同时定期汇总 `笔记数据` 和 `线索回传` 形成 `结算数据`。
- **`期望结果`**: 客户需求被完整承接，项目顺利启动，并在服务全程中有清晰、统一的沟通渠道，最终完成商业闭环。
- **`验收标准`**:
    - [ ] `项目` 对象已创建，其 `状态` 为 `待启动`，且 `关联客户` 字段不为空。
    - [ ] `项目.交付经理` 字段已更新为 `测试服务中心` 的负责人。
    - [ ] 至少存在一条状态为 `交付经理验收` 的 `里程碑事件` 记录。
    - [ ] `结算数据` 的 `状态` 已更新为 `客户已确认`。

## 场景二: 内部策略规划与验收

- **`场景ID`**: `MATRIX-PLAN-001`
- **`场景名称`**: 内部策略规划与验收
- **`优先级`**: `P0`
- **`主要角色`**: `测试服务中心`, `策划业务中心`
- **`初始情境`**: `测试服务中心` 已接收到一个 `项目` 对象，其 `状态` 为 `待启动`。
- **`核心任务与过程`**:
  1.  `测试服务中心` 对项目进行评估，创建一个 `项目策略动作` 对象，指令 `策划业务中心` 进行策略制定。
  2.  `策划业务中心` 响应指令，创建 `内容卡片`，其 `状态` 从 `策略提出` 推进至 `策略打样`。
  3.  完成策略后，`策划业务中心` 进一步创建 `测试单元` 对象，关联 `内容卡片`、`钩子胚` 和 `选题目录`。
  4.  `策划业务中心` 将 `测试单元` 提交给 `测试服务中心`，后者审核通过后，将 `测试单元.状态` 更新为 `已定级`。
- **`期望结果`**: `项目` 的宏观需求被转化为具体、可执行的 `测试单元`，并获得内部共识。
- **`验收标准`**:
    - [ ] 至少创建了一个 `内容卡片` 对象，其 `状态` 为 `Pass` 或 `可使用`。
    - [ ] 至少创建了一个 `测试单元` 对象，并成功关联了 `内容卡片`。
    - [ ] `测试单元.状态` 字段已更新为 `已定级`。

## 场景三: 测试单元落地与跟踪

- **`场景ID`**: `MATRIX-EXEC-001`
- **`场景名称`**: 测试单元落地与跟踪
- **`优先级`**: `P0`
- **`主要角色`**: `策划业务中心`, `任务排期中心`, `账号运营中心`
- **`初始情境`**: 一个 `测试单元` 的 `状态` 为 `已定级`。
- **`核心任务与过程`**:
  1.  `策划业务中心` 为该 `测试单元` 创建详细的 `内容生产计划` 和 `发布计划` 对象。
  2.  `策划业务中心` 将这两个计划对象移交给 `任务排-期中心`。
  3.  `任务排期中心` 代理跟踪，将 `内容生产计划.状态` 和 `发布计划.状态` 从 `已排期` 推进至 `执行中`，并监督 `账号运营中心` 使用 `账号资产` 进行具体操作。
- **`期望结果`**: 已通过验收的 `测试单元` 被准确、高效地执行，并产出 `笔记数据`。
- **`验收标准`**:
    - [ ] `内容生产计划` 和 `发布计划` 对象已创建，并关联了对应的 `测试单元`。
    - [ ] `发布计划.状态` 已更新为 `已完成`。
    - [ ] 至少有一条 `笔记数据` 记录通过 `发布计划` 关联回该 `测试单元`。

## 场景四: 人力筹备

- **`场景ID`**: `MATRIX-HR-001`
- **`场景名称`**: 项目人力筹备与调配
- **`优先级`**: `P0`
- **`主要角色`**: `测试服务中心`, `人力调度中心`
- **`初始情境`**: `测试服务中心` 已确认 `项目` 的整体规划和 `测试单元` 的排期。
- **`核心任务与过程`**:
  1.  `测试服务中心` 根据规划，汇总所需人力，并向 `人力调度中心` 发出请求。
  2.  `人力调度中心` 响应请求，可能创建一个 `账号小组（团）`，并为其分配 `HRBP` 和 `制作助理`、`发布助理` 等 `人员信息`。
  3.  `人力调度中心` 将调配结果（如 `账号小组ID`）同步给 `测试服务中心`。
- **`期望结果`**: `项目` 或 `测试单元` 所需的人力资源得到及时、准确的匹配和供给。
- **`验收标准`**:
    - [ ] `人力调度中心` 已收到来自 `测试服务中心` 的明确人力需求。
    - [ ] 针对项目需求，已创建 `账号小组（团）`，并为其成功关联了 `HRBP` 和 `制作助理`。
    - [ ] `测试服务中心` 已在对应的 `测试单元` 中，成功关联了 `部署账号小组` 字段。
