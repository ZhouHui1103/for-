
# ADHD督导业务 L3 - 飞书多维表格框架

**版本**: 1.0
**日期**: 2025-09-23
**创建人**: Gemini

---

## 概述

本文档基于《ADHD督导业务 L2 - 核心元数据定义 (新版).md》和《ADHD督导业务 L2 - 流程架构 (新版).md》设计，旨在为飞书多维表格的搭建提供一个完整、可执行的框架。该框架包含了支持核心业务流程所需的12张数据表，并详细定义了每张表的字段、类型及关联关系。

---

## 数据库表结构

### 1. 客户信息表
*   **概述**: 记录所有客户的基础信息，是业务的起点。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **客户ID** | **文本** | **(主字段)** 客户的唯一标识符。 |
| **状态** | **单选** | 新客, 服务中, 复购, 已流失 |
| **需求属性** | **多选** | 学习型, 工作型, 生活型 |
| **身份属性** | **单选** | 学生, 职场人士, 自由职业者 |
| **需求模块** | **多行文本** | 用于详细记录客户的具体需求。 |
| **关联场域(服务群)** | **URL** | 存放客户专属服务群的链接。 |
| **关联订单** | **关联** | 关联到 订单信息表，允许关联多个。 |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表，允许关联多个。 |
| **创建时间** | **创建时间** | 自动记录客户信息的创建时间。 |

### 2. 产品清单表
*   **概述**: 定义所有可售卖的服务产品，作为配置数据使用。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **产品名称** | **文本** | **(主字段)** 产品的名称，如30天深度督导服务。 |
| **产品ID** | **文本** | 产品的唯一编码。 |
| **服务周期** | **单选** | 7天, 14天, 30天 |
| **服务强度** | **单选** | 基础版, 标准版, 深度版 |
| **服务内容** | **多行文本** | 详细描述该产品包含的服务项目。 |
| **价格** | **货币** | 产品售卖价格。 |
| **状态** | **单选** | 设计中, 已发布, 已下架 |
| **关联订单** | **关联** | 关联到 订单信息表，查看购买此产品的订单。 |

### 3. 订单信息表
*   **概述**: 记录客户的购买行为，是服务流程启动的触发器。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **订单ID** | **自动编号** | **(主字段)** 订单的唯一编号。 |
| **客户** | **关联** | 关联到 客户信息表，选择下单的客户。 |
| **购买产品** | **关联** | 关联到 产品清单表，选择购买的产品。 |
| **订单类型** | **单选** | 初次, 复购, 好评赠送 |
| **来源渠道** | **单选** | 官网, 微信, 推荐 |
| **状态** | **单选** | 待分配, 待交付, 交付中, 交付结束 |
| **是否已给评价** | **复选框** | 用于标记客户是否已完成服务评价。 |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表，一个订单通常对应一个服务进程。 |
| **下单时间** | **创建时间** | 自动记录订单的创建时间。 |

### 4. 服务进程跟踪表
*   **概述**: 核心流程跟踪表，记录每一次客户服务的完整生命周期。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **服务进程ID** | **自动编号** | **(主字段)** 服务进程的唯一ID。 |
| **关联订单** | **关联** | 关联到 订单信息表。 |
| **客户** | **查找** | 从关联的订单信息中自动带出客户信息。 |
| **督导小队** | **关联** | 关联到 督导小队表，分配负责的小队。 |
| **状态** | **单选** | 待开启, 进程中, 已结束 |
| **服务类型** | **单选** | ADHD督导, 个人成长咨询 |
| **开启时间** | **日期** | 服务的正式开始日期。 |
| **计划结束时间** | **日期** | 根据产品周期和请假情况计算得出的预计结束日期。 |
| **实际结束时间** | **日期** | 服务最终完成的日期。 |
| **关联待办** | **关联** | 关联到 最小待办单元表，查看所有相关任务。 |
| **关联每日进度** | **关联** | 关联到 每日计划进度表，查看每日跟进记录。 |
| **关联请假记录** | **关联** | 关联到 客户请假记录表，查看客户的所有请假情况。 |
| **关联质量评估** | **关联** | 关联到 服务进程质量评估表，查看所有质量评估报告。 |

### 5. 督导小队表
*   **概述**: 人力资源单元，用于管理和分配督导资源。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **小队名称** | **文本** | **(主字段)** 如早班A队。 |
| **培训师** | **关联** | 关联到 人员信息表 (筛选角色为培训师)。 |
| **督导师列表** | **关联** | 关联到 人员信息表 (筛选角色为督导师)，允许关联多个。 |
| **状态** | **单选** | 生效, 失效 |
| **服务时段** | **单选** | 早班, 晚班, 全天 |
| **总容量** | **数字** | 该小队能同时负责的服务进程上限。 |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表，查看当前负责的服务。 |
| **已负责数量** | **汇总** | 自动计算关联服务进程的数量。**公式**: COUNT(关联服务进程) |
| **剩余容量** | **公式** | 自动计算剩余可用容量。**公式**: {总容量} - {已负责数量} |

### 6. 人员信息表
*   **概述**: 记录所有内部员工信息。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **姓名** | **文本** | **(主字段)** 员工姓名。 |
| **人员ID** | **文本** | 员工的唯一ID。 |
| **角色** | **单选** | 支撑运营, 培训师, 督导师 |
| **状态** | **单选** | 在职, 离职 |
| **工作时段** | **单选** | 早班, 晚班, 全天 |
| **所在小队** | **关联** | 关联到 督导小队表，查看所属小队。 |

### 7. 最小待办单元表
*   **概述**: 记录服务过程中需要完成的原子任务。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **待办内容** | **文本** | **(主字段)** 具体的任务描述。 |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表。 |
| **客户** | **查找** | 从关联的服务进程中自动带出客户信息。 |
| **责任人(督导师)** | **关联** | 关联到 人员信息表 (筛选角色为督导师)。 |
| **待办类型** | **单选** | 周期性默认, 客户临时提出 |
| **内容分类** | **单选** | 起床, 洗漱, 每日计划, 其他 |
| **状态** | **单选** | 待完成, 已完成 |
| **提出时间** | **创建时间** | 任务的创建时间。 |
| **督导时间** | **日期时间** | 计划执行此任务的时间。 |
| **完成时间** | **日期时间** | 实际完成此任务的时间。 |

### 8. 每日计划进度表
*   **概述**: 每日跟进客户计划完成情况的记录。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **记录ID** | **自动编号** | **(主字段)** |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表。 |
| **客户** | **查找** | 从关联的服务进程中自动带出客户信息。 |
| **记录人(督导师)** | **关联** | 关联到 人员信息表 (筛选角色为督导师)。 |
| **日期** | **日期** | 记录的日期。 |
| **评估等级** | **单选** | A, B, C |
| **备注** | **多行文本** | 对当日进度的详细文字描述。 |

### 9. 运营进程表
*   **概述**: 记录在关键节点由运营发起的标准化流程。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **运营任务** | **文本** | **(主字段)** 如XXX客户的Onboarding流程。 |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表。 |
| **客户** | **查找** | 从关联的服务进程中自动带出客户信息。 |
| **负责人(运营)** | **关联** | 关联到 人员信息表 (筛选角色为支撑运营)。 |
| **运营类型** | **单选** | Onboarding, 获取反馈, 结束服务 |
| **状态** | **单选** | 待操作, 已操作 |
| **触发时间** | **创建时间** | 任务的创建时间。 |
| **完成时间** | **日期时间** | 任务的实际完成时间。 |

### 10. 客户主观反馈表
*   **概述**: 集中记录客户在服务过程中的所有主观反馈。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **反馈ID** | **自动编号** | **(主字段)** |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表。 |
| **客户** | **查找** | 从关联的服务进程中自动带出客户信息。 |
| **反馈类型** | **单选** | Onboarding反馈, 定期反馈, 结束反馈 |
| **评分** | **评分** | 1-5分。 |
| **反馈内容** | **多行文本** | 客户反馈的具体文字内容。 |
| **反馈时间** | **创建时间** | 反馈的提交时间。 |

### 11. 客户请假记录表
*   **概述**: 记录客户在服务期间的所有请假申请与处理。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **请假事由** | **文本** | **(主字段)** 客户请假的简要原因。 |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表。 |
| **客户** | **查找** | 从关联的服务进程中自动带出客户信息。 |
| **请假类型** | **单选** | 临时事务, 身体不适, 其他 |
| **状态** | **单选** | 申请中, 已批准 |
| **请假开始时间** | **日期** | |
| **请假结束时间** | **日期** | |
| **请假天数** | **公式** | 自动计算请假时长。**公式**: DATE_DIFF({请假结束时间}, {请假开始时间}) |
| **处理人(运营)** | **关联** | 关联到 人员信息表 (筛选角色为支撑运营)。 |

### 12. 服务进程质量评估表
*   **概述**: 记录培训师对服务进程的定期评估。

| 字段名称 | 字段类型 | 选项 / 说明 |
| :--- | :--- | :--- |
| **评估ID** | **自动编号** | **(主字段)** |
| **关联服务进程** | **关联** | 关联到 服务进程跟踪表。 |
| **客户** | **查找** | 从关联的服务进程中自动带出客户信息。 |
| **评估人(培训师)** | **关联** | 关联到 人员信息表 (筛选角色为培训师)。 |
| **评估周期** | **单选** | 每周, 每月 |
| **评估时间范围** | **日期范围** | 本次评估覆盖的时间段。 |
| **评估得分** | **数字** | 1-100分。 |
| **评估意见** | **多行文本** | 具体的改进建议和评价。 |
| **评估时间** | **创建时间** | 评估报告的提交时间。 |
