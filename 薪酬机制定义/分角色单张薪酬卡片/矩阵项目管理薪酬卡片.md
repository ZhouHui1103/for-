# 矩阵项目管理薪酬卡片

*   **版本**: V1.0
*   **更新日期**: 2025-10-13
*   **文档说明**: 本文档基于《薪酬卡片文档写作规范与模板》及《L2-129.md》的核心流程，为矩阵项目管理角色提供清晰、可执行的薪酬方案。

---

### 一、 指标定义

*   **项目策略动作创建量**: 在一个考核周期内，由项目经理创建并被交付经理验收的“项目策略动作”任务总数。数据来源于`任务排期`表。
*   **内容卡片动作创建量**: 在一个考核周期内，由项目经理创建并被交付经理验收的“内容卡片动作”任务总数。数据来源于`任务排期`表。
*   **内容生产计划创建量**: 在一个考核周期内，由项目经理创建并下达给制作助理的“内容生产计划”任务总数。数据来源于`任务排期`表。
*   **发布计划创建量**: 在一个考核周期内，由项目经理创建并下达给发布助理的“发布计划”任务总数。数据来源于`任务排期`表。

---

### 二、 薪酬元规则 (Compensation Meta-rules)

#### **`XC-4-2`：矩阵项目管理任务排期薪酬规则**
1.  **规则**：根据项目经理在一个周期内完成的、不同类型的“任务排期”动作数量，分别计算其薪酬。
2.  **概念定义**：
    *   `任务排期动作`：指在《L2-129.md》中定义的，由项目管理角色发起并记录在`任务排期`表中的一系列指令性任务，主要包括`项目策略动作`、`内容卡片动作`、`内容生产计划`和`发布计划`等。
3.  **参数**：
    *   `参数1`：项目策略动作创建费（元/次）。
    *   `参数2`：内容卡片动作创建费（元/次）。
    *   `参数3`：内容生产计划创建费（元/次）。
    *   `参数4`：发布计划创建费（元/次）。
4.  **计算方法**：
    *   `薪酬 = (参数1 * 项目策略动作创建量) + (参数2 * 内容卡片动作创建量) + (参数3 * 内容生产计划创建量) + (参数4 * 发布计划创建量)`
5.  **数据来源**：`任务排期`表
6.  **相关注释**：此规则旨在通过对不同价值的管理动作进行差异化定价，来衡量项目经理的核心工作量和贡献。

---

### 三、 薪酬卡片 (Salary Cards)

#### **矩阵项目管理 薪酬卡片**
*   **项目**：矩阵号项目
*   **角色**：项目管理 / 交付经理

**1. 薪酬构成**：
*   **任务排期薪酬**: 引用元规则 `XC-4-2`，参数设定如下：
    *   `参数1` = `100` 元/次
    *   `参数2` = `50` 元/次
    *   `参数3` = `20` 元/次
    *   `参数4` = `10` 元/次

**2. 考核规则**：
*   **项目健康度**: 负责的项目不能出现重大延期或核心客户投诉。
*   **里程碑达成**: 每月至少完成1次由“交付经理/熙康”验收的“里程碑事件”。数据来源于`里程碑事件`表。
*   **项目利润**: 负责项目的最终结算利润率需大于15%（此项为激励奖金的考核前置条件，具体奖金规则另定）。数据来源于`项目财务报表`。

---

### 四、 核心指标与薪酬测算

本章节旨在基于已定义的薪酬规则，对矩阵项目管理岗位的月度薪酬进行合理性测算。

#### **1. 单节点月度到手薪酬测算**

*   **项目管理到手薪酬** = 
    ( `项目策略动作创建量` × 100 ) + 
    ( `内容卡片动作创建量` × 50 ) + 
    ( `内容生产计划创建量` × 20 ) + 
    ( `发布计划创建量` × 10 )
    
    *   **示例**: 一位项目经理在一个月内，其工作量如下：
        *   创建了 `2` 次高阶的【项目策略动作】。
        *   创建了 `5` 次【内容卡片动作】进行策略落地。
        *   创建了 `20` 次【内容生产计划】以安排素材制作。
        *   创建了 `40` 次【发布计划】以安排内容分发。
    *   **薪酬计算**:
        *   ( 2 × 100 ) + ( 5 × 50 ) + ( 20 × 20 ) + ( 40 × 10 ) 
        *   = 200 + 250 + 400 + 400
        *   = **1250元**
    *   **说明**：此为基于核心工作量计算的基础薪酬，不包含基于项目利润的激励奖金。
