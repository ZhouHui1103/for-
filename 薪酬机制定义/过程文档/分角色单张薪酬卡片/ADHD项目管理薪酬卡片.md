# ADHD项目管理薪酬卡片

*   **版本**: V1.0
*   **更新日期**: 2025-10-13
*   **文档说明**: 本文档基于《薪酬卡片文档写作规范与模板》及ADHD督导业务L2系列文档，为ADHD项目管理（支撑运营）角色提供清晰、可执行的薪酬方案。

---

### 一、 指标定义

*   **订单分配量**: 在一个考核周期内，支撑运营成功完成“1.0 订单承接与分配”流程，将一个新订单转化为一个可执行的`服务进程`的次数。
*   **客户请假处理量**: 在一个考核周期内，支撑运营成功完成“4.0 客户请假处理”流程的次数。
*   **客户反馈触达量**: 在一个考核周期内，支撑运营成功触发并完成“6.0 客户反馈收集”流程的次数。
*   **服务结束处理量**: 在一个考核周期内，支撑运营成功完成“7.0 服务续费与结束”流程，将服务进程归档并释放资源的次数。

---

### 二、 薪酬元规则 (Compensation Meta-rules)

#### **`XC-4-3`：ADHD运营管理薪酬规则**
1.  **规则**：根据支撑运营在一个周期内完成的、不同类型的标准化管理动作的数量，分别计算其薪酬。
2.  **概念定义**：
    *   `标准化管理动作`：指在《ADHD督导业务 L2 - 流程架构》中定义的，由`支撑运营`角色主导的一系列流程，主要包括`订单承接与分配`、`客户请假处理`、`客户反馈收集`和`服务续费与结束`。
3.  **参数**：
    *   `参数1`：订单分配费（元/单）。
    *   `参数2`：客户请假处理费（元/次）。
    *   `参数3`：客户反馈触达费（元/次）。
    *   `参数4`：服务结束处理费（元/次）。
4.  **计算方法**：
    *   `薪酬 = (参数1 * 订单分配量) + (参数2 * 客户请假处理量) + (参数3 * 客户反馈触达量) + (参数4 * 服务结束处理量)`
5.  **数据来源**：`订单信息表`、`客户请假记录表`、`运营进程表`、`服务进程跟踪表`
6.  **相关注释**：此规则旨在通过对不同复杂度的管理动作进行差异化定价，来衡量支撑运营岗位的核心工作量。

### 三、 薪酬卡片 (Salary Cards)

#### **ADHD项目管理 (支撑运营) 薪酬卡片**
*   **项目**：ADHD督导业务
*   **角色**：项目管理 / 支撑运营

**1. 薪酬构成**：
*   **运营管理薪酬**: 引用元规则 `XC-4-3`，参数设定如下：
    *   `参数1` (订单分配) = `30` 元/单
    *   `参数2` (请假处理) = `10` 元/次
    *   `参数3` (反馈触达) = `10` 元/次
    *   `参数4` (服务结束) = `20` 元/次

**2. 考核规则**：
*   **服务按时启动率**: `服务进程`的创建与交接必须在`订单信息`状态更新为“交付中”后的1个工作日内完成，月度达标率 > 98%。
*   **流程执行准确率**: 所有负责流程的执行错误（需返工或人工修正）次数每月不得超过 3 次。
*   **客户满意度 (NPS)**: 关联项目的整体客户满意度需维持在指定水平之上（具体数值另定）。

### 四、 核心指标与薪酬测算

本章节旨在基于已定义的薪酬规则，对ADHD项目管理岗位的月度薪酬进行合理性测算。

#### **1. 单节点月度到手薪酬测算**

*   **项目管理 (支撑运营) 到手薪酬** = 
    ( `订单分配量` × 30 ) + 
    ( `客户请假处理量` × 10 ) + 
    ( `客户反馈触达量` × 10 ) +
    ( `服务结束处理量` × 20 )
    
    *   **示例**: 一位支撑运营专员在一个月内，其工作量如下：
        *   成功分配了 `30` 个新客户订单。
        *   处理了 `15` 次客户请假。
        *   在不同阶段触发了 `50` 次客户反馈收集。
        *   完成了 `25` 个服务的结束和归档流程。
    *   **薪酬计算**:
        *   ( 30 × 30 ) + ( 15 × 10 ) + ( 50 × 10 ) + ( 25 × 20 )
        *   = 900 + 150 + 500 + 500
        *   = **2050元**
    *   **说明**：此为基于核心工作量计算的基础薪酬，不包含基于项目整体利润或客户满意度等指标的额外绩效奖金。
