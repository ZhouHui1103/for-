# 标准化流程搭建SOP (v3)

本文档旨在提供一个以“业务对象”为核心，从顶层设计、系统推演到部署验证的标准化作业流程（SOP）。

## 1. 核心设计哲学与基本要素

我们的组织搭建和流程设计遵循一个核心哲学：**一切始于业务对象**。业务对象是业务的本质，它决定了我们需要什么样的流程、数据、文档，以及由谁（即“节点”）来执行。

整个系统由三个基本要素构成：
- **场域 (Domain)**: 业务发生的上下文或场景。
- **对象 (Object)**: 业务流程中操作和管理的核心实体，是所有设计的起点。
- **节点 (Node)**: **一个统一且分层的概念**，指代流程中执行操作的最小单元。它可以是一个具体的人、一个AI Agent、一个小组、乃至一个部门。**所有后续的文档和表格设计，都必须体现节点的层级关系**。
- **流程 (Process)**: 驱动对象状态发生变化的一系列有序活动。

## 2. 系统化推导逻辑

我们的流程设计强调逻辑性和可推演行。一部分核心概念需要人为进行顶层设计，而大部分产出物都可以基于这些顶层设计系统化地推导出来。

### 阶段一：人为顶层设计 (Foundational Input)
1.  **梳理核心业务故事 (`文档`)**: 从价值主张出发，描述业务要解决的核心问题。
2.  **定义核心业务对象 (`文档`+`表格`)**: 识别出业务故事中最核心的名词，作为初始的业务对象。
3.  **划分业务节点 (`文档`)**: 对业务单元进行分层，定义出不同层级的“节点”。

### 阶段二：系统化推演 (Systematic Derivation)
1.  **详细对象设计 (`表格`+`文档`)**:
    - **推导**: 基于初始对象，根据《业务对象设计规范》详细设计其**属性、状态、类别、关联关系**等。
    - **产出**: 完整的对象清单和设计文档。

2.  **流程与自动化设计 (`Excel`+`文档`)**:
    - **推导**: 对象的“状态”变化，天然地定义了所需要的“流程”。
    - **产出**:
        - **《流程与自动化设计总表》(`Excel`)**: 这是设计的核心枢纽。在一个Excel中，系统化地设计、管理和整合三类流程及其对应的三类自动化。所有流程都应采用分层编号，以体现其所属的节点层级。
        - **L3流程文档 (`文档`)**: 对总表中的每个业务流程，撰写详细的操作、审核、指引文档。

3.  **节点与权限设计 (`表格`+`文档`)**:
    - **推导**: 基于对象和流程，可以清晰地推导出需要哪些“节点”来执行操作，以及这些节点需要什么样的“权限”。
    - **产出**: 节点手册、权限设计清单。

4.  **数据与报表设计 (`文档`+`表格`+`看板`)**:
    - **推导**: 对象的“属性”和“状态”决定了我们需要跟踪哪些数据和评估指标。
    - **产出**: 字段说明清单、报表与看板设计文档。

## 3. 流程与自动化的逻辑关系

在《流程与自动化设计总表》中，我们需要明确三类流程之间的关系：

-   **数据流程 (Data Process)**: 这是最底层的流程，定义了数据如何被创建、读取、更新和删除（CRUD）。它是业务流程能够运行的基础。
-   **业务流程 (Business Process)**: 这是中间层，描述了为达成特定业务目标，各个节点如何协作和操作“数据”。业务流程消费和产出数据，其执行依赖于底层的数据流程。
-   **报表流程 (Reporting Process)**: 这是最上层，定义了如何从业务流程产生的数据中，通过聚合、计算和分析，生成用于决策支持的报表和看板。

这三者是层层递进的关系：**数据流程支撑业务流程，业务流程产生的数据驱动报表流程。** 对应的自动化也应遵循这个逻辑，优先实现数据层面的自动化，再到业务操作，最后到报表生成。

## 4. 标准化执行流程

### 阶段一：模块定义与规划
1.  **梳理分层节点清单 (`表格`+ `文档`)**: 定义分层的业务节点。
2.  **梳理业务故事与对象 (`文档`)**: 产出核心业务故事和初始对象清单。
3.  **定义计费与考核标准 (`文档`+`表格`)**: 设计与节点价值挂钩的激励机制。
4.  **估算财务模型 (`表格`)**: 进行初步的成本收益分析。

### 阶段二：本体论构建与系统设计
1.  **设计核心对象 (`表格`+`文档`)**: 按照《业务对象设计规范》进行详细设计。
2.  **设计流程与自动化总表 (`Excel`)**: 建立多层级的流程与自动化架构。
3.  **设计数据与权限 (`表格`+`图示`+`文档`)**: 设计ER图、权限表和报表需求。
4.  **撰写L3文档 (`文档`)**: 完善每个业务流程的细节。

### 阶段三：部署与自动化
1.  **部署表格与看板系统**: 基于设计，搭建并部署完整的线上`表格`+`看板`系统。
2.  **部署自动化流程**: 利用n8n等工具实现自动化。

### 阶段四：执行部署与验证
1.  **人力流程试运行**: 通过人工完整执行一遍标准流程，验证设计的合理性。
2.  **部署跟踪 (`表格` 或项目管理工具)**: 监控流程执行情况。
3.  **财务核算验证 (`表格`+`文档`)**: 通过实际数据验证财务模型。

## 5. 核心产出物清单 (按推导顺序)

1.  **顶层设计 (人为输入)**
    - 业务故事 (`文档`)
    - 分层节点清单 (`表格`+`文档`)
    - 初始对象清单 (`表格`)
2.  **系统设计 (推导产出)**
    - 完整对象设计文档 (`表格`+`文档`)
    - 《流程与自动化设计总表》 (`Excel`)
    - L3流程文档 (`文档`)
    - 节点手册 (`文档`)
    - 权限设计清单 (`表格`)
    - 财务/薪酬/考核标准 (`文档`+`表格`)
    - 报表/看板设计文档 (`文档`)
    - 字段说明清单 (`表格`)
    - 对象关系图/流程架构图 (`图示`)

## 6. 项目管理与协作

### 6.1. 实施优先级
- **P0 (最高优)**: 业务故事 + 节点定义 + 初始对象 + 财务/薪酬定义。
- **P1 (次优)**: 详细对象设计 + 核心流程设计 + 表格系统搭建。
- **P2 (后续)**: 报表设计 + 自动化部署。

### 6.2. 团队协作模式
- **关系**: 后台组件团队负责提供标准化的模块设计（本SOP），前沿部署团队负责在具体业务中落地。
- **协同**: 细节由业务负责人完成，基础工具由 `ZH` 提供，系统验收由 `Silicon` 负责。

### 6.3. 模块跟踪与Issue管理
- **标准模块搭建**: 跟踪标准模块（如“矩阵”）从0到1的开发过程。
    - **部署标准卡片**: 作为“标准模块搭建”的最终产出，提供标准化的部署手册，用于指导后续客户部署。
- **客户部署跟踪**: 以单个客户为单位进行管理，记录其具体的部署进展和个性化调整。

### 6.4. 代码与文档管理 (Github)
- **`/tool-packages`**: 存放通用工具包。
- **`/module-packages`**: 按模块存放（主文件、检查卡片、标准日志）。
- **`/service-logs`**: 存放具体项目的服务跟踪日志。
